What is a Profile Registry?

Learn how decentralized profiles govern metadata compliance and structure open-science discovery.

W3C Profiles (prof:Profile)

A W3C profile is a semantic specification that constrains, extends, or combines standard base vocabularies (e.g. DCAT or PROV) for custom domains. It establishes machine-readable structural agreements for data communities.

Read W3C Specification

Recursive Crawler (wrx)

Our registry utilizes the high-performance wrx traversal package to recursively crawl sub-catalogs and resource links. It automatically harvests profiles and schema alignments while filtering noisy blank nodes to preserve clean graphs.

W3C PROV-O Provenance

Every registration automatically generates a Dublin Core and W3C PROV-O provenance chain, stored inside a dedicated named graph (#provenance). This records the parsed source URL, discovery pathways, requesting issue, and pull request.

Automated Registration Workflow

Here is how our decentralized pipeline recursively processes and indexes profile submissions.

01
Submit URI
Provide a persistent profile or catalog URI (e.g. PurL, w3id, URN) into our registration form below.
02
Trigger Run
A structured GitHub Issue is generated. Our automated CI workflow instantly boots up to resolve and crawl your resource.
03
Parse & Clean
The crawl resolves recursive nodes via wrx, isolates profile predicates, filters blank nodes, and seals W3C PROV-O provenance.
04
PR & Publish
An automated Pull Request is opened. Upon administrative approval, the profile enters the globally aggregated registry CSV & N-Quads.
